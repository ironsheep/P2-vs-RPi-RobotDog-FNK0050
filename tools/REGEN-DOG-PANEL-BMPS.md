# Regenerate the Move-Panel BMP assets (host-native)

How to regenerate the five `dog_*.bmp` panel assets from
`tools/gen_dog_panel_assets.py`. **Run this on the host (macOS), not in the
dev container** — the generator renders text with the system fonts
(`Helvetica.ttc` / `Arial.ttf` / `SFNS.ttf`); if those are missing it silently
falls back to an ugly bitmap default font. The container has no Pillow and no
those fonts, so the committed BMPs are always Mac-generated.

You are running host-native, so you can run these tools directly.

## What gets written

The script writes **5 files into `src/`** (it computes `OUT = ../src`):

| File | What it is |
|------|------------|
| `dog_panel_bg.bmp` | background: title, all buttons, readout labels + black value boxes, footer |
| `dog_btn_hi.bmp` | vertical strip of highlighted button cells |
| `dog_font.bmp` | readout digit/sign font strip (`0123456789-`) |
| `dog_modes.bmp` | robot mode-name word strip (IDLE/GAITING/GESTURE/RELAXED/LOWBATT) |
| `dog_led_modes.bmp` | **NEW** — LED-mode word strip (OFF/SOLID/WIPE/CHASE/RAINBOW/CYCLE) |

It also prints a **Spin2 `CON` block** to stdout that must match the generated
layout constants pasted into `src/test_dog_panel.spin2`.

## Steps

```bash
# 1. From the repo root
cd "$(git rev-parse --show-toplevel)"

# 2. Make sure Pillow is available (one-time)
python3 -c "import PIL; print('Pillow', PIL.__version__)" \
  || python3 -m pip install --user Pillow

# 3. Generate the BMPs (and capture the emitted CON block)
python3 tools/gen_dog_panel_assets.py | tee /tmp/dog_panel_cons.txt
```

Expected tail of the output:

```
wrote: .../src/dog_panel_bg.bmp
wrote: .../src/dog_btn_hi.bmp
wrote: .../src/dog_font.bmp
wrote: .../src/dog_modes.bmp
wrote: .../src/dog_led_modes.bmp
panel 660x510; button cell 100x40 ...
' ---- layout (generated by ...) ----
  PANEL_W = 660
  PANEL_H = 510
  ...
```

## Verify before flashing

```bash
# 4. All five BMPs present, dog_led_modes.bmp is the new one
ls -l src/dog_panel_bg.bmp src/dog_btn_hi.bmp src/dog_font.bmp \
      src/dog_modes.bmp src/dog_led_modes.bmp

# 5. What changed (expected: dog_led_modes.bmp NEW, dog_panel_bg.bmp modified
#    — it grew a row and gained the LED readout box; the other three usually
#    re-emit byte-identical)
git status --short src/dog_*.bmp
```

**6. Confirm the emitted CONs match `src/test_dog_panel.spin2`.** The `.spin2`
already carries values that should equal the generator's output. Sanity-check
the LED additions in particular:

```
  PANEL_H    = 510
  LED_CELL_W = 150
  LED_CELL_H = 30
  RO_LED_BX  = 71
  RO_LED_BY  = 426
  RO_LED_BW  = 178
  RO_LED_S0  = 227
```

If any emitted value differs from what's in `src/test_dog_panel.spin2`, paste
the generator's `CON` block over the matching constants there (the script is
the single source of truth) and re-check that the `setupWindow()` window-size
literal reads `size 660 510`.

**7. Flash.** With the BMPs regenerated, flash `src/test_dog_panel.spin2` to the
P2 (2 Mbaud) and run bench exercise **Ex 6**: step the LED button and confirm
the on-screen readout shows the active mode name + index (OFF→SOLID→WIPE→CHASE
→RAINBOW→CYCLE→OFF) in step with the ring.

## Commit

The regenerated BMPs are tracked assets — commit them alongside the firmware
change (commit directly to `main`; Stephen signals when). Include the new
`src/dog_led_modes.bmp`.
